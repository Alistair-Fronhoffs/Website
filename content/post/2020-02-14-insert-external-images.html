Sidestep: Including external images I need to include some screenshots and this should be easy peasy, but I have spent quite some time wrestling with this. Up untill then I was knitting the posts .rmd file, to check it’s html output. However inserting the image gave me an error (“file not found”). Because I’ve got this .rdm file in the website folder as a new post, I don’t have to knit it anymore but check the post in my Viewer where it is automatically updated every time I save the file ::smiley::

So, just to show you, blogdown comes with addins, and I just had to choose “insert Image” in the addins menu.
It will insert code in the form of:
![alternative text](path to file, starting from your /static folder)
Blogdown assumes the path is stated from your static folder Or through Tools > Addins > Browse Addins:

And then click the ‘Insert Image’ addin:

Another possibility is using knitr’s function
knitr::include_graphics()
The basic form would be
knitr::include_graphics(file to path starting from \static\)

I heavily relied on Alison Presmanes Hill’s minimal blogdown website, so do check it out for more information !
